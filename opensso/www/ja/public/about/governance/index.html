OpenSSO Governance Proposal - Draft

The OpenSSO project follows a simple governance structure. Decisions are made in public discussion groups by consensus. All decisions are made transparently and any member of the project may voice their concerns in the discussion.

Everyone is welcome to join the OpenSSO community and make contributions. OpenSSO members follow the code of conduct and abide by the governance process. They may play one or more of the following roles in the community.

User

All OpenSSO project members are Users. Users use the software produced by the community. They file issues, defects, and requests for enhancements and new features. They share their experiences in using the software with others in the community by publishing articles, and contributing to forums and wikis. You become a user by creating an account on java.net and joining the community. Request the Observer role which is the equivalent to the User role.

Contributor

Contributors are Users who contribute source code or documentation changes to resolve issues, make feature enhancements, or incorporate new functionality. Contributors do not have write access to the source tree. To become a Contributor, complete and sign the Sun Contributor Agreement (SCA). Once the SCA is registered, all Users will be notified. If your role is not approved within two working days, forward your registration note to sca@opensso.dev.java.net. Request the Observer role which, when coupled with a signed SCA, is equivalent to the Contributor role.

Committer

Committers are Developers with write access to the source tree. Committers review patches submitted by Developers. Once a patch is approved by the committers and module owners, it will be applied to the source tree by one of the Committers. Most Committers only have write privileges to the portion of the source tree that they are most familiar with. Developers are considered for the Committer role based on the quality of their contributions to the project. The work must show an expertise in one or more of the OpenSSO modules, demonstrated through patches made to the source tree, and contributions to the forums, or wiki. Request the Developer role which is equivalent to Committer.

Module Owner

Module Owners are Committers responsible for reviewing and maintaining the quality of a specific module. They merge patches and new code from the development branch of the source tree into the stable branch of the particular module. They are responsible for making sure that these contributions do not break the build by performing a second level review as well as an architecture review of the proposed changes. Module owners are appointed by creating a consensus among Committers and Module Owners. Module Owners are initially appointed by Sun. Request the Developer role and Content Developer role.

Project Managers

Project Managers are responsible for managing the entire OpenSSO project, helping to create policies by consensus that ensure global quality. Project Managers are responsible for creating the project branch of the source tree. Branches are created for sustaining, major enhancements, and experiments. Project Managers are also responsible for making any updates to the governance policy. All decisions are made by consensus. In cases where a consensus can not be reached, Project Managers make the final decision. Project Managers are appointed by Sun. Request the Developer, Content Developer and Project Owner roles.
